Binary Search

Overview Visualization Implementation Application

Binary search is the most popular search algorithm.

It is one of the most commonly used techniques, and is also one of the most efficient.

It uses a 'divide and conquer' style to search for elements with an O(log n) time complexity.

This algorithm is usually implemented recrusively, but can easily be implemented iteratively as well.

As useful as binary search is, it only works on a collection of sorted elements.


Algorithm

A low, mid, and high point are defined and used to help find the target element.

If the target element is not at the mid point, we check to see if it is higher or lower than the midpoint.

If the target element is lower than the element at the mid point, we move the high point to be one less than the mid point.

If the target element is higher than the elemenet at the mid point, we move the low point to be one more than the mid point.


This method allows us to effectively cut the size of elements that we search through in half after every iteration.
